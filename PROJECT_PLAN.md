## 🎯 專案願景

建立一個 **即時、可視化、多人協作的 AI 辦公室儀表板**，讓老闆能夠：
- 👀 隨時看到所有 AI 助手的工作狀態
- 🏢 在多個不同風格的場景中切換
- 🗣️ 與 AI 助手互動
- 📊 了解實驗室和研究團隊的運作情況

---

## 📌 功能優先順序

### 老闆專區

老闆希望能夠 **「放自己的實際形象進去」**：

#### Option A: 像素風頭像
- 用 AI 生成老闆的像素風頭像
- 可以放在場景中的特定位置
- 有多種表情（工作、休息、說話）

#### Option B: 實際照片
- 老闆上傳一張照片
- 自動調整為適合的大小
- 可以選擇要不要用濾鏡

#### Option C: AI 生成虛擬形象
- 用 AI 生成一個符合老闆風格的虛擬角色
- 可以自訂服裝、髮型、配件

- 🌸 粉色頭髮的女生

## 🔧 技術架構

### 技術堆疊

| 層 | 技術 | 說明 |
|----|------|------|
| 前端遊戲 | Phaser.js 3 | 像素風格遊戲引擎 |
| 前端 UI | HTML/CSS/JS | 控制面板 |
| 後端 | Flask (Python) | API 伺服器 |
| 資料儲存 | JSON 檔案 | 狀態和設定 |

### 檔案結構

```
aiOfficeUI/
├── backend/
│   ├── app.py              # Flask 後端
│   ├── store_utils.py      # 資料儲存
│   └── memo_utils.py       # 備忘錄功能
├── frontend/
│   ├── index.html          # 主頁面
│   ├── game.js             # 遊戲邏輯
│   ├── layout.js           # 場景配置
│   └── assets/             # 圖片資源
├── agents-state.json       # AI 狀態
├── state.json              # 主要狀態
├── join-keys.json          # 驗證金鑰
└── todo.md                 # 工作清單
```

### API 端點

| 端點 | 方法 | 用途 |
|------|------|------|
| `/` | GET | 首頁 |
| `/status` | GET | 取得主要狀態 |
| `/agents` | GET | 取得所有 AI |
| `/set_state` | POST | 設定狀態 |
| `/agent-push` | POST | 推播 AI 狀態 |
| `/static/*` | GET | 靜態資源 |
