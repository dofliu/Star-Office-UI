## 目標 1：即時代理人狀態儀表板（P0）

### 現有基礎
- ✅ 主代理人（Star）狀態透過 `state.json` + `GET /status` 即時渲染
- ✅ 訪客代理人透過 `POST /agent-push` 推送狀態，前端透過 `GET /agents` 輪詢
- ✅ 狀態到區域映射（idle→休息區、writing→工作區、error→Bug角落）
- ✅ `ai-office-bridge.py` 支援 Claude Code / Gemini CLI / Codex CLI 橋接

### 待開發
| 任務 | 說明 | 複雜度 |
|------|------|--------|
| 1.1 進度條顯示 | 在角色氣泡或側邊面板顯示 `progress` 百分比（後端已支援欄位） | 低 |
| 1.2 工作內容面板 | 點擊代理人角色彈出詳情面板，顯示 `detail`、`state`、`progress`、`updated_at` | 中 |
| 1.3 代理人列表側邊欄 | 左側或右側可摺疊側邊欄，列出所有在線代理人及其即時狀態 | 中 |
| 1.4 狀態推送增強 | bridge 腳本增加更豐富的 detail（檔名、命令摘要、子任務描述） | 低 |
| 1.5 即時通知 | 代理人狀態變化時顯示 toast 通知（如：「Claude Code 開始工作」） | 低 |
| 1.6 WebSocket 升級 | 將輪詢改為 WebSocket/SSE 推送，降低延遲與流量 | 高 |

## 目標 2：代理人工作歷史紀錄（P1）

### 現有基礎
- ✅ `yesterday-memo` API 提供昨日備忘錄（來自 memory 目錄）
- ✅ `agents-state.json` 記錄當前所有代理人狀態
- ❌ 沒有歷史狀態持久化機制

### 待開發
| 任務 | 說明 | 複雜度 |
|------|------|--------|
| 2.1 狀態歷史 API | 新增 `GET /agent-history/<agentId>` 返回該代理人的歷史狀態紀錄 | 中 |
| 2.2 狀態日誌持久化 | 每次狀態變更時 append 到 `agent-logs/` 目錄下的 JSONL 日誌檔 | 中 |
| 2.3 歷史面板 UI | 前端點擊代理人 → 展開時間軸，顯示最近 N 筆狀態變更 | 中 |
| 2.4 每日摘要 | 根據狀態日誌自動產生每日工作摘要（工作時長、狀態分佈等） | 高 |
| 2.5 日誌清理策略 | 自動清理超過 N 天的日誌，避免無限增長 | 低 |
| 2.6 搜尋功能 | 依日期、代理人、關鍵字搜尋歷史紀錄 | 中 |

## 目標 3：像素畫面客製化（P1）

### 現有基礎
- ✅ 背景圖可透過 Asset Editor 替換（`office_bg_small.webp`）
- ✅ Gemini API 可生成新房間背景（`/generate-room`）
- ✅ 背景收藏夾功能（`home-favorites/`）
- ✅ 傢俱位置可透過 `asset-positions.json` 調整
- ✅ 訪客有 6 種不同的動畫角色（`guest_anim_1~6`）

### 待開發
| 任務 | 說明 | 複雜度 |
|------|------|--------|
| 3.1 場景套裝系統 | 預設多套場景主題（辦公室、咖啡廳、太空站等），一鍵切換 | 高 |
| 3.2 畫布解析度設定 | 允許使用者調整畫布大小（如 1920×1080、800×600），layout.js 動態縮放 | 中 |
| 3.3 人物選擇器 | UI 面板讓使用者選擇代表自己的角色（從可用的 sprite 中選擇） | 中 |
| 3.4 訪客角色偏好 | 訪客加入時可指定偏好角色，`/join-agent` 增加 `preferredAvatar` 欄位 | 低 |
| 3.5 自訂角色上傳 | 允許上傳自訂 sprite sheet 作為角色（需規範格式） | 高 |
| 3.6 深色/淺色模式 | 辦公室背景與 UI 支援深色模式切換 | 中 |
| 3.7 縮放控制 | 前端加入 zoom in/out 按鈕，動態縮放像素畫面 | 低 |

## 技術決策原則

1. **保持原生 JS + Phaser** — 不引入前端框架（遵循 CLAUDE.md）
2. **JSON 檔案優先** — 輕量場景用 JSON 持久化，必要時才考慮 SQLite
3. **漸進式增強** — 每個功能獨立可用，不造成破壞性變更
4. **三語支援** — 新 UI 文字需同時提供中/英/日翻譯
5. **Desktop 相容** — 確保 Tauri/Electron 封裝不受影響

## 如何在 Desktop Claude 繼續開發

在本機 clone 此 repo 後，可直接接續開發：

```bash
# 拉取最新分支
git fetch origin claude/analyze-project-docs-BUR8S
git checkout claude/analyze-project-docs-BUR8S

# 安裝依賴
python3 -m pip install -r backend/requirements.txt

# 啟動開發伺服器
cd backend && python3 app.py

# 開始開發（參考本文件的 Phase 順序）
```

開發時請參考：
- `CLAUDE.md` — 專案規範與禁止事項
- `docs/ROADMAP.md` — 本藍圖（你正在看的這份）
- `docs/DEVELOPMENT.md` — 技術架構與開發指南
